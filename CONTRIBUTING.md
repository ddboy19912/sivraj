# Contributing

## Working Agreement

Sivraj is early. Optimize for learning speed while preserving the core product thesis:

> Persistent intelligence, not generic memory.

Before adding features, ask whether the work improves:

- Continuity.
- Synthesis.
- Agent coordination.
- User-owned context.
- Compounding personalization.

## Development Flow

1. Pick a task from [TODO](./docs/TODO.md).
2. Keep changes scoped.
3. Update docs when product behavior changes.
4. Add tests for risky or shared logic.
5. Record major product or architecture decisions in [DECISIONS](./docs/DECISIONS.md).

## Documentation Rules

- Use `Sivraj` as the product name.
- Use `Twin` for the user's evolving cognitive model.
- Avoid positioning the product as a chatbot, notes app, memory vault, or vector database.
- Explain infrastructure components by the product role they serve.

## Commit Style

Recommended commit prefixes:

- `docs:`
- `feat:`
- `fix:`
- `refactor:`
- `test:`
- `chore:`
