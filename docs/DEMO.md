# Demo Plan

## Demo Narrative

Do not pitch Sivraj as AI memory.

Pitch:

> A persistent intelligence layer across all AI systems.

## Demo Goal

Make the user feel that Sivraj understands personal and strategic patterns across their work better than any single AI tool could.

## Demo Flow

### 1. Upload Life and Work Data

Use a prepared founder dataset containing:

- GitHub project history.
- Product notes.
- Investor notes.
- PDFs.
- Voice note transcript.
- Chat export.
- Weekly planning notes.

Show that Sivraj accepts messy real-world context.

### 2. Build Identity Graph Live

Show:

- People.
- Goals.
- Projects.
- Decisions.
- Patterns.
- Relationships between them.

The graph should feel like a living model of the user, not a file browser.

### 3. Use Multiple AI Systems

Show three agent views:

- Coding agent.
- Research agent.
- Strategy agent.

Each agent receives different scoped context from the same Twin.

### 4. Wow Moment

Ask:

> What patterns consistently slow my progress?

The answer should be:

- Personal.
- Strategic.
- Specific.
- Evidence-backed.
- Emotionally real.

Example response:

> You repeatedly delay launches by over-optimizing UI polish after core functionality is already working. This appeared across the wallet dashboard, investor portal, and onboarding redesign. The pattern usually begins after external feedback creates ambiguity, then shifts your effort from shipping to polishing.

### 5. Close the Loop

Show that the insight can become:

- A goal update.
- A weekly reminder.
- A strategy note.
- A context rule for future agents.

## Demo Assets Needed

- Sample founder dataset.
- Sample identity graph.
- Three simulated agent clients.
- One excellent synthesis response.
- One permission screen.
- One context packet inspector.

## Demo Anti-Patterns

Avoid showing:

- A generic chatbot.
- A file storage screen as the main value.
- Raw embeddings.
- Storage pricing.
- A notes app interface.
