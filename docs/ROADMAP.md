# Roadmap

This roadmap describes the public product direction for Sivraj. It avoids internal decision history and implementation notes.

## Current Focus

Sivraj is focused on the core loop:

1. Create or resume a wallet-backed Twin.
2. Add private context.
3. Process source material safely.
4. Ask questions over memory and documents.
5. Return source-backed answers.
6. Share scoped context with approved tools and agents.

## Near Term

- Improve onboarding and first memory creation.
- Make upload and processing states clearer.
- Strengthen source-grounded answers for documents and saved memory.
- Expand memory review and correction flows.
- Harden provider configuration and failure states.
- Publish stable API examples for context retrieval.
- Improve local development and deployment documentation.

## Integrations

- JavaScript SDK.
- Python SDK.
- CLI workflows for developers.
- MCP-compatible tool access.
- Coding-agent context packets.
- Desktop capture and review.
- Mobile capture and reflection.

## Infrastructure

- More explicit processing lifecycle states.
- Better retry and recovery behavior for background jobs.
- Clearer permission scopes for external clients.
- Safer context packet contracts.
- More complete observability for ingestion and retrieval.

## Longer Term

- User-managed integration marketplace.
- Team and workspace memory boundaries.
- Advanced permission review.
- Cross-tool context continuity.
- Portable memory exports.
