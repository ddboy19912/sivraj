# User Stories

## Founder OS

- As a founder, I want to upload product notes, GitHub history, investor notes, and planning docs so Sivraj can understand my operating context.
- As a founder, I want to ask what is slowing my progress so I can see recurring execution bottlenecks.
- As a founder, I want weekly strategic reflection so I can adjust priorities before drift becomes expensive.
- As a founder, I want my coding agent to know architecture history so I do not repeat context every session.
- As a founder, I want my strategy agent to see investor and roadmap context without seeing unrelated private memories.

## AI-Native Knowledge Worker

- As a knowledge worker, I want Sivraj to remember my workflows so AI tools can help me faster.
- As a knowledge worker, I want my communication preferences to follow me across AI systems.
- As a knowledge worker, I want to retrieve past decisions with citations so I can avoid re-deciding the same things.
- As a knowledge worker, I want synthesis across projects so I can notice patterns I would otherwise miss.

## Agent Builder

- As an agent builder, I want an API for scoped user context so my agent can personalize behavior safely.
- As an agent builder, I want context packets with citations so my agent can explain where its assumptions came from.
- As an agent builder, I want permission scopes so users can trust my integration.

## Security-Conscious User

- As a user, I want to decide which agents can access which memories.
- As a user, I want to inspect what context an agent received.
- As a user, I want to revoke access at any time.
- As a user, I want sensitive memories encrypted and private by default.

## Foundation Acceptance Criteria

- A user can create a Twin.
- A user can add or upload memory sources.
- Sivraj extracts retrievable memory fragments.
- Sivraj builds a basic graph of goals, projects, people, and decisions.
- A user can ask a synthesis question and receive an evidence-backed answer.
- Different agent scopes produce different context packets.
